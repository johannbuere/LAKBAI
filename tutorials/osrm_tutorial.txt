NOTE: -> MAKE SURE TO PUT THE DIRECTORY AT THE C DRIVE FOR THIS TUTORIAL TO WORK. Example: C:\LAKBAI\osrm-docker
	  -> WE will be creating multiple profiles for each routing (e.g car, foot, bicycle) so make sure you have the following directories:
		For car profile: C:\LAKBAI\osrm-docker\car
		For foot profile: C:\LAKBAI\osrm-docker\foot
		For bicycle profile: C:\LAKBAI\osrm-docker\bicycle
      

Prerequisites before tutorial:
-> Download albay.osm.pbf from LAKBAI repo(repo natin). Make sure it's in all the directories (folders: car, bicycle, and foot)
	example: C:\LAKBAI\osrm-docker\car\albay.osm.pbf
-> Have Docker downloaded and setup


OSRM Installation Process for Windows
(Optional) Visit https://github.com/Project-OSRM/osrm-backend for premade code and documentation

CAR PROFILE (INITIAL)

Step 1: Go or make a directory named LAKBAI at C drive
	Example C:\LAKBAI\osrm-docker\car

Step 2: Move albay.osm.pbf to  C:\LAKBAI\osrm-docker\car directory. MAKE SURE TO BE IN THE C:\LAKBAI\osrm-docker\car DIRECTORY WHEN DOING THE FOLLOWING CODES. 
	NOTE: just do: cd  C:\LAKBAI\osrm-docker\car
 
Step 3: Paste this code to Windows Powershell: docker run -t -v "${PWD}:/data" ghcr.io/project-osrm/osrm-backend osrm-extract -p /opt/car.lua /data/albay.osm.pbf

Step 4: Paste this code next to Windows Powershell: docker run -t -v "${PWD}:/data" ghcr.io/project-osrm/osrm-backend osrm-partition /data/albay.osrm

Step 5: Next, past another code again: docker run -t -v  C:\LAKBAI\osrm-docker\car:/data ghcr.io/project-osrm/osrm-backend osrm-customize /data/albay.osrm

Step 6: Check server if it's up and running on localhost: docker run -t -i -p 5000:5000 -v "${PWD}:/data" ghcr.io/project-osrm/osrm-backend osrm-routed --algorithm mld /data/albay.osrm

	-> Note: Make sure code line ends with: "[info] running and waiting for requests"
	-> DO NOT CLOSE THIS TERMINAL. CHECK UNTIL THE END
	

Step 7 (Final): Open a new terminal and past -> curl "http://127.0.0.1:5000/route/v1/driving/13.388860,52.517037;13.385983,52.496891?steps=true"

	-> Note -> Make sure code status shows OK

CLOSING TERMINALS: Go back to the terminal you used at Step 6 and press Ctrl+C. This ends the service. DO NOT DELETE ANYTHING IN DOCKER DASHBOARD. ONLY PAUSE ONE FOR EACH PORT (E.G. PORT 5000:5000, OR 5000:5001). DO NOT STOP DOCKER AS WELL
